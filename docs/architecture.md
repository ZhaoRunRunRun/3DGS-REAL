# 3DGS REAL 架构说明

## 1. 客户端职责

Flutter 客户端负责：

- 引导拍摄
- 图像导入与预处理
- 参数配置
- 本地任务调度
- 远端任务提交与状态跟踪
- 模型结果展示

## 2. 推荐模块划分

### `lib/pages`
- 页面级 UI：首页、采集页、任务列表页、任务详情页、模型预览页

### `lib/widgets`
- 复用控件：状态卡片、步骤条、角度覆盖图、照片质量面板

### `lib/services`
- 业务服务：相机服务、相册服务、任务服务、上传服务、模型预览服务

### `lib/models`
- 数据模型：重建任务、照片元信息、建模参数、导出产物

## 3. 建模流程建议

1. 采集图像
2. 执行质量检查
3. 做相机位姿恢复（如 COLMAP）
4. 启动 3DGS 训练
5. 导出结果并提供预览

## 4. 本地模式建议

更适合 Windows/macOS/Linux：

- Flutter 发起本地命令或调用本地服务
- Python 负责 COLMAP 与 3DGS 训练
- Flutter 监听日志并展示训练状态

## 5. 远端模式建议

更适合 Android：

- Flutter 负责压缩/上传照片
- 服务端负责队列调度、训练、产物存储
- Flutter 轮询或订阅任务状态

## 6. 推荐服务端技术栈

- Python + FastAPI
- COLMAP 用于 SfM / MVS
- 3D Gaussian Splatting 实现（如 gsplat 或官方实现）
- 对象存储保存图片和建模结果
- Redis / Celery 或类似队列系统管理任务

## 7. 后续扩展

- AR 预览
- 训练参数模板
- 多对象项目管理
- 多人协作与远端共享
